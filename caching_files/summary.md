## Итог по кешированию:

1. Большие файлы не оптимально (теряется производительность на моменте записи в кеш, переполнение оперативки).
2. Нет смысла кеширования в Redis при обычных SELECT-операциях (shared buffers у Postgres справляется сам).
3. Хороший пример: [Видео нагрузочного тестирования с Redis и без него](https://www.youtube.com/watch?v=QpBaA6B1).
4. Кеширование хорошо себя покажет при наличии агрегирующих функций, какого-то подсчёта перед ответом.
5. В проекте получился контрпример, когда Redis не дает никакого прироста, а делает даже хуже
